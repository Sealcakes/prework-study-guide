HTML

  • HTML was first written by Tim Berners-Lee in 1993
  • An element is a type of content on a webpage
  • Elements are also used to divide the content on the page and to describe the content to browsers and other developers.
  • The head element contains information about the webpage.
  • The head element contains metadata or machine-readable infromation.
  • The body element represents the visible content shown to the user.
  • You use different html tags to build the framework of your webpage.
  • The doctype element defines the document type, in this case we are using HTML.

Git

  • Git is a version control system that lets you manage and keep track of each version of your project.
  • To access Git, we use the command line (Git Bash on Windows)
  • GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories that are stored remotely (in the cloud)
  • A Repository (Repo) is a project on Github, a folder for your project that contains all of the files and their revision history.
  • A Branch is a workspace where you can make changes that don't affect the live site.
  • Markdown is the way content files are formatted in GitHub, Markdown files use the .md file extension.
  • Committing Changes is how you save a new version of a file you are working on, whenever you add, edit, or delete a file, you're making a commit.
  • Pull Requests are asking for changes on a branch to merge into another branch.
  • Merging is when a Pull Request is approved, changes are "pulled in" or merged into the main branch.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
